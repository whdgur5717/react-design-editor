# Discuss Implementation

GitHub Issue를 기반으로 구현 방향을 논의합니다.

## 워크플로우

1. Issue 내용 확인
2. 관련 코드베이스 탐색
3. 구현 접근법 제안
4. 사용자와 대화형 논의
5. (선택) 합의 내용을 Issue 코멘트로 기록

## 실행 방법

### 1. Issue 내용 확인

```bash
gh issue view {number}
```

### 2. 코드베이스 탐색

- 관련 파일 구조 파악
- 기존 패턴/컨벤션 확인
- 영향받는 코드 범위 분석

### 3. 논의 진행

다음 항목들을 사용자와 논의합니다:

- **기술 선택**: 어떤 라이브러리/패턴을 쓸지
- **파일 구조**: 어떤 파일을 만들고/수정할지
- **인터페이스**: API/타입 설계
- **엣지케이스**: 고려해야 할 예외 상황
- **테스트 전략**: 어떻게 검증할지

### 4. 논의 결과 기록 (선택)

```bash
gh issue comment {number} --body "## 구현 방향

### 결정 사항
- ...

### 수정할 파일
- ...

### 고려사항
- ..."
```

## 입력

- GitHub Issue 번호 (필수)

## 출력

- 합의된 구현 방향
- 수정할 파일 목록
- 구현 순서

## 다음 단계

논의가 완료되면 `/start-work {issue번호}`로 작업을 시작하세요.
