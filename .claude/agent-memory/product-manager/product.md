# Product Definition

## Why it exists

Design tools like Figma create a gap: you design something, then need a separate process to turn it into code. Often this involves AI translation (Figma MCP, etc.), which introduces inaccuracy and an unnecessary middleman. The question is: why should AI be needed in this process at all?

This editor eliminates that gap. The editor renders real React components — what you see IS the code. No conversion step required.

## Vision

Replace Figma. A design editor where the output is production React code, with no intermediary process.

## Target User

Anyone who uses a design editor. The distinction between "designer" and "developer" is outdated — in the AI era, the person designing the UI is the same person who ships the code.

## User Pain Points

- Figma → Code requires a translation step (manual or AI), which loses design intent
- AI-based design-to-code tools add an unnecessary dependency to a process that shouldn't need it
- The current design → code workflow has too many handoff points

## Core Value Proposition

Design = Code. The editor renders real React components, so there's no conversion step needed. What you edit is what you ship.

## Competitive Landscape

| Tool       | Gap                                                                              |
| ---------- | -------------------------------------------------------------------------------- |
| Figma      | Design tool only — needs AI/plugin to generate code, loses intent in translation |
| Framer     | Closer to code but proprietary output, not standard React                        |
| Builder.io | AI-dependent code generation from designs                                        |

Our differentiator: the editor renders actual React components. There is no translation layer. The design IS the code.
